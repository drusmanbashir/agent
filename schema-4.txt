Current Defects

2. Preprocessing is gated by client follow-up. train_plan_ready returns breakpoint=preproc instead of letting the orchestrator submit preproc, observe briefly, and continue when possible.
3. provider/model are recorded but not used as a real Ollama client config.
4. Status payloads are not canonical. Status strings and fields vary across train/preproc/job paths.

Target Architecture
Flow:
HTTP request -> validate web TrainRequest -> build train command + OrchestratorConfig -> LangGraph orchestrator -> assess -> decide -> act -> observe -> debug/repair if needed -> repeat -> terminal StatusResult.

The LangGraph graph lives in the orchestrator layer, not in web.py, service.py, tools, or FRAN code. Ollama is used at decision nodes when state is ambiguous, when natural-language intent must become a tool call, or when failure triage needs classification. Deterministic routing is preferred when state is clear, such as plan preprocessed=both -> submit_train, missing preproc -> submit_preproc, active job -> observe_job, known failed job -> build_crash_packet.

/api/orchestrator/requests/train remains the public endpoint. Its semantics change from gated single-shot to a capped autonomous loop. It resolves prerequisites autonomously where safe, but once train is successfully submitted it normally returns status=submitted rather than waiting through a long train runtime.

HTTP Lifetime: Path A
The autonomous loop runs synchronously inside run_in_threadpool and is capped by OrchestratorConfig.max_iterations, max_poll_attempts, poll_interval_seconds, and deadline_seconds.
The cap is for prerequisite resolution and short observation only. It can wait briefly for preproc/job state transitions, but must not hold the HTTP request for a full training run.
If preproc or another prerequisite is still active when the cap is reached, return status=timed_out with active job_id, log_root, poll_url, and last_observation.
If train submission succeeds, return status=submitted with train job_id, log_root, poll_url, stdout/stderr URLs, and crash packet URL.
timed_out is terminal for the HTTP request, not proof that underlying work failed. The caller polls existing endpoints such as /api/local-train/jobs/{job_id}, /api/local-train/jobs/{job_id}/crash, stdout, and stderr.

LangGraph Orchestrator Contract
Graph state contains request command, config, observations, selected tool call, active job, crash packet, repair attempts, and terminal StatusResult.
Nodes:
assess_plan: read project/plan/preproc state.
decide_next: deterministic route when state is clear; Ollama JSON decision only when ambiguous.
submit_preproc: submit local/HPC preproc through tools.
observe_prereq: short poll for preproc or prerequisite jobs.
submit_train: submit train and return submitted on success.
observe_train_short: optional brief observation only to catch immediate launch failure.
debug_failed_job: build crash packet and classify failure.
escalate_codex: call Codex adapter when enabled and retry policy allows.
rerun_failed_stage: rerun failed stage after repair.

Failed Job Debugging
Failure classes:
launch_failure: worker process, missing executable, bad cwd, missing env, permission error.
preproc_failure: analyze/resample command failed or expected preprocessed artifacts absent after completion.
train_failure: train_retry exited non-zero, immediate import/config error, CUDA/device error, data loading error, OOM, or metric/runtime crash.
registry_failure: job row missing, stale job, missing job_dir, missing stdout/stderr, malformed metadata.
infra_failure: /s unavailable, HPC submit/poll failure, dashboard unavailable, log root unwritable.

Crash packet shape:
job_id, stage, status, failure_class, error_code, message, command, cwd, env_summary, job_dir, log_root, stdout_path, stderr_path, stdout_tail, stderr_tail, worker_log_tail, poll_log_tail, job_meta, worker_meta, orch_json, dashboard, timestamps, previous_attempts, suspected_owner, recommended_action.
Persist packet at <job_dir>/crash_packet.json and expose /api/local-train/jobs/{job_id}/crash.

Triage loop:
observe failed status -> build crash packet -> deterministic classify if signatures are clear -> Ollama classify only when ambiguous -> choose retryable, codex_repairable, blocked, or terminal failed.
Retry policy:
retry transient infra/registry observation once when no code/config change is needed.
retry resource-shaped failures only through bounded parameter change if already supported by train_retry policy.
do not retry deterministic code/config failures without repair.
Terminal debug statuses:
debugging while packet/classification is being built.
repair_submitted when Codex repair has been delegated.
debug_failed when triage/repair attempts are exhausted.
failed when the workflow reaches an unrepaired terminal failure.

Codex Escalation
Add escalate_to_codex(packet) as an adapter at the orchestrator boundary. It receives only the crash packet plus minimal repo paths and returns repair_result with status, branch, commit, pr_url, summary, and validation.
Branch naming: codex/<job_id>/<attempt>.
Attempt cap: up to 3 repair attempts per job/stage.
For each repair attempt:
create or update branch codex/<job_id>/<attempt>.
apply the smallest code/config/test repair needed for the packet.
rerun the failed stage after fix through the existing tool path.
if rerun succeeds, resume the LangGraph workflow from the failed node.
if rerun fails, build a new crash packet and either attempt the next repair or terminally fail.
After 3 failed attempts, return debug_failed or failed with packet paths and next_action for user review.
On successful repair, assign branch/PR to the user for merge/review and include branch and pr_url in StatusResult.details. The orchestrator does not auto-merge.

Validation
Normal path: preprocessed plan returns submitted train with job_id, poll_url, stdout/stderr URLs, crash URL, and log_root.
Autonomous prerequisite path: missing preproc submits preproc, observes within cap, reassesses, then submits train if ready before deadline.
Timed-out prerequisite path: long preproc returns timed_out with active job identity and polling links.
Immediate train failure path: train launch fails quickly, crash_packet.json is written, failure_class is set, and debug status is returned.
Codex repair path: a synthetic repairable failure calls escalate_to_codex(packet), creates codex/<job_id>/<attempt>, reruns failed stage, and resumes on success.
Repair exhaustion path: three failed attempts return debug_failed with packet paths and next_action.
Recovery path: /s/agent_rw/hpc_logs unavailable writes registry/artifacts under ~/.agent/hpc_logs and job polling still resolves.
Determinism: repeated requests with matching active work observe existing jobs instead of duplicating submissions.
